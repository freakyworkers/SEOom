# 포인트 교환 기능 테스트 체크리스트

## 관리자 기능 테스트

### 1. 관리자 로그인 및 접근
- [ ] 관리자 계정으로 로그인 (admin@test.com / test1234)
- [ ] 관리자 패널에서 "포인트 교환" 메뉴 확인
- [ ] 포인트 교환 관리 페이지 접근 확인

### 2. 설정 관리
- [ ] 페이지 제목 설정 (예: "포인트교환")
- [ ] 공지사항 제목 설정 (예: "포인트 교환 필독 사항 안내")
- [ ] 안내 항목 추가/삭제 기능 테스트
- [ ] 최소값/최대값 설정
- [ ] 교환 신청 폼 필드 추가 (예: 은행명, 계좌번호 등)
- [ ] 설정 저장 확인

### 3. 상품 관리
- [ ] 상품 추가 버튼 클릭
- [ ] 썸네일 없이 상품 등록 (항목명: "사이트", 항목내용: "에그벳", 공지 입력)
- [ ] 상품 목록 테이블에 등록된 상품 확인
- [ ] 상품 수정 기능 테스트
- [ ] 상품 삭제 기능 테스트

### 4. 신청 목록 관리
- [ ] 특정 상품의 "신청목록" 버튼 클릭
- [ ] 신청 목록 페이지 접근 확인
- [ ] 신청자 정보, 신청금액, 폼 데이터 확인
- [ ] 상태 변경 (대기 → 완료, 대기 → 보류 등)
- [ ] 보류 상태일 때 거절사유 입력 및 저장
- [ ] 상태 변경 후 포인트 차감/환불 확인

## 사용자 기능 테스트

### 5. 사용자 로그인 및 접근
- [ ] 사용자 계정으로 로그인 (aa@test.com / aaaaaaaa)
- [ ] 포인트 교환 페이지 접근 (/point-exchange)
- [ ] 페이지 제목 및 공지사항 확인

### 6. 교환 신청
- [ ] 교환 상품 목록 확인
- [ ] 특정 상품의 "교환하기" 버튼 클릭
- [ ] 상품 상세 페이지 확인:
  - [ ] 썸네일 표시 (없으면 이미지 아이콘)
  - [ ] 항목명, 항목내용 표시
  - [ ] 처리 현황 (완료 건수, 보류 건수, 완료 금액)
  - [ ] 보유 포인트 표시
- [ ] 교환 포인트 입력
- [ ] 신청 폼 필드 입력 (은행명, 계좌번호 등)
- [ ] 신청 버튼 클릭
- [ ] 신청 완료 메시지 확인
- [ ] 포인트 차감 확인

### 7. 신청 내역 확인
- [ ] 신청 내역 테이블 확인
- [ ] 신청일, 신청인, 신청포인트, 상태 확인
- [ ] 대기 상태일 때 "취소" 버튼 표시 확인
- [ ] 취소 버튼 클릭 및 확인 다이얼로그
- [ ] 취소 후 포인트 환불 확인
- [ ] 보류 상태일 때 물음표 아이콘 표시 확인
- [ ] 물음표 아이콘 호버 시 거절사유 툴팁 표시 확인

### 8. 상태별 테스트
- [ ] 관리자가 상태를 "완료"로 변경
- [ ] 사용자 페이지에서 상태가 "완료"로 표시되는지 확인
- [ ] 관리자가 상태를 "보류"로 변경하고 거절사유 입력
- [ ] 사용자 페이지에서 상태가 "보류"로 표시되고 물음표 아이콘 확인
- [ ] 물음표 아이콘 호버 시 거절사유 확인
- [ ] 보류 시 포인트 환불 확인

## 포인트 관리 테스트

### 9. 포인트 차감/환불 로직
- [ ] 신청 시 포인트 차감 확인
- [ ] 취소 시 포인트 환불 확인
- [ ] 보류 시 포인트 환불 확인
- [ ] 완료 → 보류 변경 시 포인트 환불 확인
- [ ] 보류 → 완료 변경 시 포인트 차감 확인

## 통계 및 집계 테스트

### 10. 상품 통계
- [ ] 상품별 대기/완료/보류 건수 확인
- [ ] 상품별 총액 확인
- [ ] 상태 변경 시 통계 업데이트 확인

## UI/UX 테스트

### 11. 반응형 디자인
- [ ] 모바일 화면에서 레이아웃 확인
- [ ] 테이블 반응형 처리 확인

### 12. 에러 처리
- [ ] 최소값/최대값 범위 밖 입력 시 에러 메시지
- [ ] 보유 포인트 부족 시 에러 메시지
- [ ] 필수 필드 미입력 시 에러 메시지

## 보안 테스트

### 13. 권한 확인
- [ ] 비로그인 사용자 접근 제한
- [ ] 일반 사용자의 관리자 페이지 접근 제한
- [ ] 다른 사용자의 신청 취소 시도 시 제한

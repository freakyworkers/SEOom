# 이벤트 게시판 테스트 체크리스트

## 테스트 환경
- URL: http://localhost:8000
- 사이트: test-site (또는 생성된 사이트)
- 관리자 계정 필요

---

## 1단계: 마이그레이션 확인 및 실행

### 확인 사항
- [ ] 이벤트 관련 마이그레이션 파일 존재 확인
  - `2025_11_25_000005_add_event_display_type_to_boards_table.php`
  - `2025_11_25_000006_add_event_fields_to_posts_table.php`
  - `2025_11_25_000007_create_event_options_table.php`
  - `2025_11_25_000008_create_event_participants_table.php`

### 실행 명령어
```bash
php artisan migrate
```

---

## 2단계: 이벤트 게시판 생성

### 테스트 시나리오
1. 관리자로 로그인
2. 관리자 페이지 → 게시판 관리
3. 게시판 생성 클릭
4. 게시판 정보 입력:
   - 이름: "테스트 이벤트 게시판"
   - Slug: "test-event-board"
   - 게시판 타입: **이벤트** 선택
   - 이벤트 표시 타입: "일반" 또는 "사진" 선택
5. 저장

### 확인 사항
- [ ] 게시판이 정상적으로 생성됨
- [ ] 게시판 타입이 "이벤트"로 표시됨

---

## 3단계: 공지형 이벤트 게시글 작성

### 테스트 시나리오
1. 생성한 이벤트 게시판으로 이동
2. "글쓰기" 버튼 클릭
3. 게시글 작성:
   - 제목: "공지형 이벤트 테스트"
   - 내용: "이것은 공지형 이벤트입니다."
   - **이벤트 유형: 공지형 이벤트** 선택
   - 시작일: 오늘 날짜
   - 종료일: 내일 날짜 (또는 종료일 미정 체크)
4. 저장

### 확인 사항
- [ ] 게시글이 정상적으로 작성됨
- [ ] 이벤트 정보가 표시됨 (유형: 공지형 이벤트)
- [ ] **참여 버튼이 없음** (공지 형태)
- [ ] 이벤트 상태가 올바르게 표시됨 (예정/진행중/종료)

---

## 4단계: 신청형 이벤트 게시글 작성 및 참여

### 테스트 시나리오 A: 게시글 작성
1. 이벤트 게시판에서 "글쓰기" 클릭
2. 게시글 작성:
   - 제목: "신청형 이벤트 테스트"
   - 내용: "이것은 신청형 이벤트입니다."
   - **이벤트 유형: 신청형 이벤트** 선택
   - 시작일: 오늘 날짜
   - 종료일: 내일 날짜
3. 저장

### 확인 사항
- [ ] 게시글이 정상적으로 작성됨
- [ ] 이벤트 정보가 표시됨 (유형: 신청형 이벤트)
- [ ] **"이벤트 신청하기" 버튼이 표시됨**

### 테스트 시나리오 B: 이벤트 신청
1. 일반 사용자로 로그인 (또는 다른 브라우저/시크릿 모드)
2. 신청형 이벤트 게시글 열기
3. "이벤트 신청하기" 버튼 클릭
4. AJAX로 처리되는지 확인 (페이지 새로고침 없이 처리)

### 확인 사항
- [ ] 버튼 클릭 시 "처리 중..." 표시
- [ ] 신청 완료 후 "이미 이벤트에 참여하셨습니다" 메시지 표시
- [ ] 참여자 수가 증가함
- [ ] **중복 클릭 방지됨** (버튼이 비활성화됨)
- [ ] 페이지 새로고침 없이 처리됨 (AJAX)

### 테스트 시나리오 C: 운영자 일괄 포인트 지급
1. 운영자 계정으로 로그인
2. 신청형 이벤트 게시글 열기
3. "운영자 기능: 일괄 포인트 지급" 섹션 확인
4. 포인트 입력 (예: 100)
5. 대상 선택 (전체 참여자 또는 미지급 참여자만)
6. "포인트 지급하기" 버튼 클릭

### 확인 사항
- [ ] 포인트가 정상적으로 지급됨
- [ ] 성공 메시지 표시 (예: "1명의 참여자에게 100포인트를 지급했습니다.")
- [ ] 참여자 기록에 포인트가 업데이트됨

---

## 5단계: 정답형 이벤트 게시글 작성 및 참여

### 테스트 시나리오 A: 게시글 작성
1. 이벤트 게시판에서 "글쓰기" 클릭
2. 게시글 작성:
   - 제목: "정답형 이벤트 테스트"
   - 내용: "이것은 정답형 이벤트입니다."
   - **이벤트 유형: 정답형 이벤트** 선택
   - 시작일: 오늘 날짜
   - 종료일: 내일 날짜
   - **퀴즈 선택지 추가:**
     - 선택지 1: "정답입니다" (정답 체크)
     - 선택지 2: "오답입니다"
     - 선택지 3: "이것도 오답입니다"
3. 저장

### 확인 사항
- [ ] 게시글이 정상적으로 작성됨
- [ ] 이벤트 정보가 표시됨 (유형: 정답형 이벤트)
- [ ] 퀴즈 선택지가 표시됨
- [ ] **"정답을 선택하세요" 폼이 표시됨**

### 테스트 시나리오 B: 정답 선택 및 시각적 피드백
1. 일반 사용자로 로그인
2. 정답형 이벤트 게시글 열기
3. 정답 선택:
   - 라디오 버튼 클릭

### 확인 사항
- [ ] **선택한 항목에 파란색 테두리와 배경색이 표시됨**
- [ ] **버튼 하단에 "선택한 정답: [정답 텍스트]" 표시됨**
- [ ] 다른 선택지를 클릭하면 이전 선택이 해제되고 새 선택이 강조됨

### 테스트 시나리오 C: 정답 제출
1. 정답 선택 후 "정답 제출하기" 버튼 클릭
2. AJAX로 처리되는지 확인

### 확인 사항
- [ ] 버튼 클릭 시 "처리 중..." 표시
- [ ] 정답인 경우: "정답입니다! 이벤트에 참여했습니다." 메시지
- [ ] 오답인 경우: "오답입니다. 이벤트에 참여했습니다." 메시지
- [ ] **선택한 정답 텍스트가 결과에 표시됨**
- [ ] 정답/오답 배지가 표시됨
- [ ] 참여자 수가 증가함
- [ ] 선택지별 참여자 수가 업데이트됨
- [ ] **중복 클릭 방지됨**

### 테스트 시나리오 D: 이미 참여한 경우
1. 이미 참여한 사용자로 같은 게시글 열기

### 확인 사항
- [ ] "이미 이벤트에 참여하셨습니다" 메시지 표시
- [ ] **선택한 정답 텍스트가 표시됨**
- [ ] 운영자 또는 이벤트 종료 시 정답/오답 여부 표시

---

## 6단계: 중복 참여 방지 테스트

### 테스트 시나리오
1. 신청형 또는 정답형 이벤트에 참여
2. 같은 페이지에서 빠르게 여러 번 버튼 클릭 시도

### 확인 사항
- [ ] 첫 번째 클릭 후 버튼이 비활성화됨
- [ ] 중복 참여가 방지됨
- [ ] 데이터베이스에 중복 레코드가 생성되지 않음

---

## 7단계: 이벤트 상태 테스트

### 테스트 시나리오
1. 시작일이 미래인 이벤트 생성 → 상태: "예정"
2. 시작일이 과거이고 종료일이 미래인 이벤트 → 상태: "진행중"
3. 종료일이 과거인 이벤트 → 상태: "종료"

### 확인 사항
- [ ] 이벤트 상태가 올바르게 표시됨
- [ ] 종료된 이벤트는 참여 버튼이 표시되지 않음

---

## 8단계: 에러 처리 테스트

### 테스트 시나리오
1. 로그인하지 않은 상태에서 참여 버튼 클릭
2. 종료된 이벤트에 참여 시도
3. 정답형 이벤트에서 정답 선택 없이 제출 시도

### 확인 사항
- [ ] 적절한 에러 메시지 표시
- [ ] 로그인 페이지로 리다이렉트 (로그인 필요 시)
